# 👥 Stakeholder Analysis & Engagement Matrix
## تحليل أصحاب المصلحة | مشروع جرين بوكس

> النجاح في تقليل التأخير بنسبة 25% لا يعتمد فقط على البيانات، بل على توافق جميع الأطراف. يهدف هذا الملف إلى تحديد القوى المؤثرة في المشروع ورسم استراتيجية لضمان دعمهم الكامل.

قُمنا بتقسيم الأطراف المعنية إلى ثلاث فئات رئيسية:

| الطرف المعني | الدور في المشروع | الفئة |
| :--- | :--- | :--- |
| **نجد الحيدان (CEO)** | راعي المشروع (Sponsor) | إداري |
| **نجود الناجم** | مدير المشروع (Green Belt) | قيادي |
| **فريق العمليات** | إدارة المستودع والفرز | تشغيلي |
| **سائقو التوصيل** | التنفيذ الميداني للميل الأخير | تشغيلي |
| **قسم التقنية (IT)** | دعم نظام التتبع والعناوين | تقني |
| **الشريك التجاري** | العملاء المتعاقد معهم | خارجي |
| **العميل النهائي** | مستلم الشحنة | خارجي |

## 2. مصفوفة القوة والاهتمام (Power/Interest Matrix)

بناءً على منهجية إدارة المشاريع، تم تصنيف الأطراف لاتخاذ استراتيجية التعامل المناسبة:

| الطرف المعني | القوة (Power) | الاهتمام (Interest) | الاستراتيجية (Strategy) |
| :--- | :--- | :--- | :--- |
| **الإدارة العليا (CEO)** | عالية | عالية | **الإدارة اللصيقة:** تقارير أسبوعية عن الـ ROI |
| **سائقو التوصيل** | منخفضة | عالية | **الإبقاء على اطلاع:** تدريب وحوافز للأداء |
| **إدارة العمليات** | عالية | عالية | **الشراكة الكاملة:** إشراكهم في تصميم الحلول |
| **قسم التقنية (IT)** | متوسطة | متوسطة | **المشاورة:** التنسيق عند تحديث الخرائط |
| **العميل النهائي** | منخفضة | عالية | **الإرضاء:** تحسين الشفافية وزمن الوصول |

## 3. مصفوفة المسؤوليات (RACI Matrix)

توضح هذه المصفوفة من هو المسؤول عن ماذا في مرحلة التعريف (Define):

| المهمة (Task) | نجود (PM) | نجد (CEO) | العمليات | السائقين | تقنية المعلومات |
| :--- | :---: | :---: | :---: | :---: | :---: |
| صياغة ميثاق المشروع | **R/A** | **I** | **C** | **I** | **I** |
| تحديد الـ CTQs للعميل | **R/A** | **I** | **C** | **C** | **I** |
| رسم مخطط الـ SIPOC | **R/A** | **I** | **C** | **C** | **I** |
| اعتماد ميزانية التحسين | **C** | **R/A** | **I** | **I** | **I** |

*   **R** (Responsible): المسؤول عن التنفيذ.
*   **A** (Accountable): المسؤول عن النتيجة النهائية (صاحب القرار).
*   **C** (Consulted): يتم استشارته قبل التنفيذ.
*   **I** (Informed): يتم إبلاغه بعد التنفيذ.

## 4. استراتيجية التواصل والمخاطر البشرية

### 🛡️ إدارة مقاومة التغيير (Resistance Management)
*   **المخطر:** قد يشعر السائقون أن نظام التتبع الجديد هو "مراقبة لصيقة" وليس "تحسين كفاءة".
*   **الحل:** ربط تقليل التأخير بحوافز مالية للسائقين الأسرع والأكثر دقة، وتوضيح أن النظام يهدف لتسهيل وصولهم للموقع وليس لمراقبتهم.

### 📢 قنوات التواصل
*   **الإدارة:** لوحة بيانات (Dashboard) لحظية توضح نسبة التحسن.
*   **السائقون:** اجتماعات "Gemba" صباحية قصيرة لمناقشة تحديات العناوين.
*   **العملاء:** إشعارات تلقائية عبر التطبيق توضح حالة الرحلة.

## 5. خريطة التأثير (Influence Map)

تأثير المشروع يتجاوز قسم التوصيل؛ حيث أن نجاح Green Box في تقليل التأخير سيؤدي إلى:
1.  **قسم المالية:** تقليل هدر ساعات العمل الإضافية بنسبة 15%.
2.  **قسم التسويق:** تحسين سمعة العلامة التجارية وزيادة نقاط التقييم في المتجر.
3.  **قسم المبيعات:** زيادة عدد الطلبات المتكررة (Retention Rate).
